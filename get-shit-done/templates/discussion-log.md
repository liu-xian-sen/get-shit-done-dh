# 讨论日志 (Discussion Log) 模板

用于 `.planning/phases/XX-name/{phase_num}-DISCUSSION-LOG.md` 的模板 —— 记录讨论阶段 (discuss-phase) 问答会话的审计轨迹。

**目的：** 为决策提供软件审计轨迹。捕获所有考虑过的选项，而不仅仅是选中的那个。与 CONTEXT.md 不同，后者是供下游代理使用的实现产物。

**不供 LLM 使用。** 此文件绝不应在 `<files_to_read>` 块或代理提示词中被引用。

## 格式

```markdown
# 阶段 [X]: [名称] - 讨论日志 (Discussion Log)

> **仅供审计。** 请勿将其作为计划、研究或执行代理的输入。
> 决策结果已在 CONTEXT.md 中捕获 —— 此日志仅保留所考虑过的备选方案。

**日期：** [ISO 日期]
**阶段：** [阶段编号]-[阶段名称]
**讨论领域：** [以逗号分隔的列表]

---

## [领域 1 名称]

| 选项 | 描述 | 已选中 |
|--------|-------------|----------|
| [选项 1] | [简要描述] | |
| [选项 2] | [简要描述] | ✓ |
| [选项 3] | [简要描述] | |

**用户选择：** [选中的选项或原始自由文本回复]
**备注：** [讨论期间提供的任何澄清或理由]

---

## [领域 2 名称]

...

---

## Claude 的裁量权

[委派给 Claude 判断的领域 —— 列出延后的内容及其原因]

## 延后想法

[提到的但不在本阶段范围内的想法]

---

*阶段: XX-name*
*讨论日志生成时间: [日期]*
```

## 规则

- 在每次讨论阶段 (discuss-phase) 会话结束时自动生成
- 包含所有考虑过的选项，而不仅仅是选中的那个
- 包含用户的自由格式备注和澄清
- 明确标记为仅供审计，不是实现产物
- 不干扰 CONTEXT.md 的生成或下游代理的行为
- 与 CONTEXT.md 在同一个 git 提交中提交
