# 路线图模板

`.planning/ROADMAP.md` 的模板。

## 初始路线图 (v1.0 Greenfield)

```markdown
# 路线图：[项目名称]

## 概览

[用一段话描述从开始到完成的历程]

## 阶段

**阶段编号：**
- 整数阶段 (1, 2, 3)：计划中的里程碑工作
- 小数阶段 (2.1, 2.2)：紧急插入的工作 (标记为 INSERTED)

小数阶段按数字顺序出现在其周围的整数阶段之间。

- [ ] **阶段 1：[名称]** - [单行描述]
- [ ] **阶段 2：[名称]** - [单行描述]
- [ ] **阶段 3：[名称]** - [单行描述]
- [ ] **阶段 4：[名称]** - [单行描述]

## 阶段详情

### 阶段 1：[名称]
**目标**：[此阶段交付的内容]
**依赖**：无 (第一阶段)
**需求**：[REQ-01, REQ-02, REQ-03]  <!-- 括号可选，解析器支持两种格式 -->
**成功标准** (必须为真)：
  1. [从用户视角可观察的行为]
  2. [从用户视角可观察的行为]
  3. [从用户视角可观察的行为]
**计划**：[计划数量，例如 "3 个计划" 或 "待定"]

计划：
- [ ] 01-01：[第一个计划的简短描述]
- [ ] 01-02：[第二个计划的简短描述]
- [ ] 01-03：[第三个计划的简短描述]

  1. [从用户视角可观察的行为]

计划：

## 进度

**执行顺序：**
阶段按数字顺序执行：2 → 2.1 → 2.2 → 3 → 3.1 → 4

| 阶段 | 计划完成情况 | 状态 | 完成日期 |
|-------|----------------|--------|-----------|
| 1. [名称] | 0/3 | Not started | - |

| 3. [名称] | 0/2 | Not started | - |
| 4. [名称] | 0/1 | Not started | - |
```

<guidelines>
**初始规划 (v1.0)：**
- 阶段数量取决于粒度设置 (粗略：3-5，标准：5-8，精细：8-12)
- 每个阶段交付连贯的内容
- 每个阶段可以有 1 个或多个计划 (如果超过 3 个任务或涉及多个子系统，请拆分)
- 计划命名规则：{phase}-{plan}-PLAN.md (例如 01-02-PLAN.md)
- 不设时间预估 (这不是企业级项目管理)
- 进度表由执行工作流更新
- 计划数量最初可以是 "待定"，在规划期间完善

**成功标准：**
- 每个阶段 2-5 个可观察的行为 (从用户视角)
- 在路线图创建期间对照需求进行交叉检查
- 在计划阶段向下流转至 `must_haves`
- 在执行后由验证阶段进行验证
- 格式："用户可以 [动作]" 或 "[事物] 正常工作/已存在"

**里程碑发布后：**
- 使用 `<details>` 标签折叠已完成的里程碑
- 为即将开展的工作添加新的里程碑章节
- 保持连续的阶段编号 (永远不要从 01 重新开始)
</guidelines>

<status_values>
- `Not started` - 尚未开始
- `In progress` - 正在进行
- `Complete` - 已完成 (添加完成日期)
- `Deferred` - 已推迟 (注明原因)
</status_values>

## 里程碑分组的路线图 (v1.0 发布后)

完成第一个里程碑后，使用里程碑分组进行重新组织：

```markdown
# 路线图：[项目名称]

## 里程碑

- ✅ **v1.0 MVP** - 阶段 1-4 (发布于 YYYY-MM-DD)
- 🚧 **v1.1 [名称]** - 阶段 5-6 (进行中)
- 📋 **v2.0 [名称]** - 阶段 7-10 (已计划)

## 阶段

<details>
<summary>✅ v1.0 MVP (阶段 1-4) - 已发布 YYYY-MM-DD</summary>

### 阶段 1：[名称]
**目标**：[此阶段交付的内容]
**计划**：3 个计划

计划：
- [x] 01-01：[简短描述]
- [x] 01-02：[简短描述]
- [x] 01-03：[简短描述]

[... 剩余的 v1.0 阶段 ...]

</details>

### 🚧 v1.1 [名称] (进行中)

**里程碑目标：**[v1.1 交付的内容]

#### 阶段 5：[名称]
**目标**：[此阶段交付的内容]
**依赖**：阶段 4
**计划**：2 个计划

计划：
- [ ] 05-01：[简短描述]
- [ ] 05-02：[简短描述]

[... 剩余的 v1.1 阶段 ...]

### 📋 v2.0 [名称] (已计划)

**里程碑目标：**[v2.0 交付的内容]

[... v2.0 阶段 ...]

## 进度

| 阶段 | 里程碑 | 计划完成情况 | 状态 | 完成日期 |
|-------|-----------|----------------|--------|-----------|
| 1. Foundation | v1.0 | 3/3 | Complete | YYYY-MM-DD |
| 2. Features | v1.0 | 2/2 | Complete | YYYY-MM-DD |
| 5. Security | v1.1 | 0/2 | Not started | - |
```

**说明：**
- 里程碑图标：✅ 已发布，🚧 进行中，📋 已计划
- 已完成的里程碑折叠在 `<details>` 中以提高可读性
- 当前/未来的里程碑保持展开状态
- 连续的阶段编号 (01-99)
- 进度表包含里程碑列
