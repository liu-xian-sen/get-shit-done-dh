# GSD 指令

- 当用户请求 GSD 或使用 `gsd-*` 命令时，使用 get-shit-done 技能 (skill)。
- 将 `/gsd-...` 或 `gsd-...` 视为命令调用，并从 `.github/skills/gsd-*` 加载匹配的文件。
- 当命令要求派生子代理时，优先使用来自 `.github/agents` 的匹配自定义代理。
- 除非用户明确要求，否则不要应用 GSD 工作流。
- 在完成任何 `gsd-*` 命令（或它触发的任何交付物：功能、错误修复、测试、文档等）之后，务必：(1) 通过 `ask_user` 提示用户下一步操作；重复此反馈循环，直到用户明确表示已完成。
